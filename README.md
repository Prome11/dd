## How To Request Help

Open a public inquiry here:

[Create a Service Inquiry](https://github.com/Prome11/dd/issues/new?template=service-inquiry.yml)

Please only share non-sensitive summaries in public issues. Do not post private company documents, personal IDs, passwords, payment information, unpublished business secrets, or customer data. Private materials should be shared only after the scope and delivery channel are confirmed.

## Delivery Process

1. You describe the goal, deadline, audience, and current material status.
2. I confirm whether the request fits a fixed package or needs a custom quote.
3. We lock deliverables, timeline, revision boundary, and safe material channel.
4. You receive the agreed output plus reusable notes, templates, or checklists.

## Operating Principles

- Real work only: no fake cases, fake income claims, fake testimonials, or misleading before/after results.
- Clear scope: every order defines deliverables, deadline, revision count, source material, and privacy boundary.
- Public-safe first: GitHub issues are only for non-sensitive inquiry summaries.
- Useful artifacts: every delivery should leave behind reusable templates, checklists, or examples.
